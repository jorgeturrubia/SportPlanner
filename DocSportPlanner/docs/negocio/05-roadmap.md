```markdown
# 5. ROADMAP

---

## Resumen ejecutivo

Este roadmap convierte las user stories priorizadas en un plan de entrega de sprints para el MVP y fases iniciales.

## Horizonte: 12 meses (MVP 5 meses)

### Objetivos principales
- Tener un MVP funcional con editor visual, planificador y visualizador
- Validar modelo de negocio con 500 entrenadores en 12 meses

## Timeline sugerido (Sprints de 2 semanas)

### Sprint 1-2: Fundamentos
- US-001 a US-003: Autenticación
- US-004 a US-007: Gestión de objetivos
- US-012: Crear planificación básica

### Sprint 3-4: Editor Visual (complejo)
- US-008 a US-011: Editor con canvas y animaciones

### Sprint 5-6: Planificador Inteligente
- US-013 a US-016: Selección de objetivos y algoritmo de progresión

### Sprint 7-8: Sesiones y Visualizador
- US-017 a US-022: Crear sesiones y modo entrenador

### Sprint 9-10: Calendario y Pulido
- US-023 a US-026: Calendario, creación rápida y Marketplace (lite)
- Bugfixes, optimizaciones y tests

### Fase 2 (Meses 6-12)
- Marketplace completo, Reportes y Director roles
- Integraciones, escalado y mejoras de producto

## Hitos
- Hito 1: MVP minimo viable (Sprint 5-6) — editor + planificador + visualizador
- Hito 2: Publicación beta y onboarding (Sprint 7-8)
- Hito 3: Version 1.0 Pública (Sprint 9-12)

## Dependencias críticas
- Definición final del Algoritmo de Progresión
- Decisión sobre librería de canvas y rendimiento (Fabric.js + GSAP)
- Contenido del Marketplace para importar

## Riesgos y mitigaciones
- Complejidad del editor visual — Mitigación: MVP con subset de features y librería probada
- Escasez de contenido en Marketplace — Mitigación: crear 50-100 ejercicios semilla

---

**Estado:** borrador — completar fechas concretas por sprint/fecha una vez priorizadas las stories
```
