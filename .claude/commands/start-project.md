# 🚀 Iniciar Nuevo Proyecto: $ARGUMENTS

Por favor, usa el agente **orchestrator** para iniciar un nuevo proyecto llamado "$ARGUMENTS".

## Instrucciones para el Orchestrator:

1. **Fase de Definición** (product-manager)
   - Crear visión del producto
   - Definir requisitos iniciales
   - Establecer casos de uso principales

2. **Fase de Diseño** (architect)
   - Diseñar arquitectura base
   - Definir patrones a utilizar
   - Crear estructura del proyecto

3. **Fase de Selección** (stack-selector)
   - Evaluar opciones tecnológicas
   - Seleccionar stack apropiado
   - Activar agentes específicos

4. **Fase de Setup** (implementer)
   - Crear estructura de carpetas
   - Configurar herramientas base
   - Generar archivos iniciales

5. **Documentación Final**
   - README.md con instrucciones
   - Documentación de inicio rápido
   - Próximos pasos claros

## Resultado Esperado:
- Proyecto completamente documentado
- Stack tecnológico definido
- Estructura lista para desarrollo
- Agentes configurados para el stack

¡Asegúrate de que todo quede documentado en la estructura `docs/`!
