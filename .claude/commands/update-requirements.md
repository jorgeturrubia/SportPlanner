# 📝 Actualizar Requisitos del Proyecto

Invocar al agente **product-manager** para actualizar o completar la documentación de requisitos.

## Tareas para el Product Manager:

1. **Revisar documentación existente**
   - Verificar @docs/product/requirements.md
   - Revisar historias de usuario actuales
   - Analizar casos de uso documentados

2. **Identificar gaps**
   - Requisitos faltantes
   - Historias incompletas
   - Casos de uso no cubiertos

3. **Actualizar documentación**
   - Agregar nuevos requisitos funcionales
   - Definir requisitos no funcionales faltantes
   - Actualizar prioridades si es necesario

4. **Validar consistencia**
   - Verificar trazabilidad
   - Confirmar que no hay contradicciones
   - Asegurar completitud

5. **Generar reporte**
   - Resumen de cambios realizados
   - Nuevos requisitos agregados
   - Impacto en arquitectura (si aplica)

El product-manager debe asegurar que toda la documentación siga los templates establecidos y mantenga consistencia con el resto del proyecto.
