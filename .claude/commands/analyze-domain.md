Analyze the business domain and generate technical specifications:

**Domain**: $ARGUMENTS

**Analysis Process**:
1. **Domain Discovery**:
   - Identify key business entities
   - Map business relationships
   - Define business rules and constraints
   - Understand user workflows

2. **Technical Translation**:
   - Generate Angular models and interfaces
   - Design .NET API endpoints
   - Create Supabase schema
   - Define validation rules

3. **UI/UX Patterns**:
   - Domain-specific components
   - Business workflow screens
   - Data visualization needs
   - User role considerations

4. **Integration Requirements**:
   - External system integrations
   - Business rule enforcement
   - Reporting and analytics
   - Performance considerations

Use business-analyst agent to perform domain analysis and coordinate with other specialists for implementation.

**Example Domains**:
- sports "Athletic training and performance management"
- billing "Invoice and payment processing system"
- ecommerce "Online retail platform"
- education "Learning management system"
- healthcare "Patient management system"
